# 《天下要闻｜A股映射日报》Tushare 数据增强模块

更新时间：2026-03-28

## 一、目标
当日报涉及相关个股、大宗商品、期货合约时，使用 Tushare 对新闻映射结果做二次验证，避免只凭新闻联想给出标的。

---

## 二、数据增强流程

### Step 1：先做新闻归因
先判断新闻主要影响哪类资产：
- 原油 / 油运 / 黄金 / 军工 / 网络安全 / 出口链 / 科技链

### Step 2：建立候选池
从新闻逻辑出发，形成候选：
- 行业板块
- 相关个股
- 相关期货 / 商品合约

### Step 3：拉取市场数据（Tushare）
优先拉取：
- A股个股日线 / 若可用则分钟级数据
- 期货合约日线 / 若可用则分钟级数据
- 交易日历与交易时段校验

### Step 4：做共振判断
至少检查以下维度：
- 最近 1 日涨跌幅
- 最近 3 日趋势
- 成交额 / 成交量变化
- 是否顺着新闻逻辑在走
- 是否存在“逻辑强、股价弱”或“消息弱、股价先走”的背离

### Step 5：输出分级
将候选分成：
- **高胜率方向**：新闻逻辑强 + 数据共振较强
- **高相关方向**：新闻逻辑强，但走势节奏一般
- **观察方向**：逻辑存在，但数据配合度不足
- **剔除方向**：缺少数据支撑，避免误导

---

## 三、默认输出字段
每个重点标的尽量输出：
- 名称 / 代码
- 最新收盘价或最新可核验价格
- 近 1 日涨跌幅
- 近 3 日走势观察
- 与主线是否共振
- 判断等级（高胜率 / 高相关 / 观察 / 剔除）
- 一句话结论

---

## 四、期货 / 商品规则
涉及大宗和期货时：
- 原油主线优先参考上海原油期货（如 SC 主力相关合约）
- 黄金主线优先参考沪金合约（如 AU 主力相关合约）
- 必须说明数据时点与合约代码
- 不把单日波动直接等同于趋势结论

---

## 五、风控要求
- 不得把延迟数据说成实时数据
- 非交易时段要明确口径（最近交易日 / 收盘数据）
- 不得因新闻强就硬推所有映射个股
- 有背离时要明确写出背离，而不是强行圆逻辑

---

## 六、默认结论结构
每次数据增强后，末尾增加：
- 更偏高胜率的方向
- 高相关但更看节奏的方向
- 本次数据增强后的结论修正
